### 4.1 日期格式转换

| API | 格式 | 示例 |
|-----|------|------|
| 本公司API | `YYYY-MM-DD` 或 `YYYY-MM-DD HH:mm:ss` | 2025-12-30 或 2025-12-30 16:59:45 |

**转换规则**：
长软 → 本公司：
- 格式转换：`YYYYMMDD` → `YYYY-MM-DD`
- 实现逻辑：提取前4位年份、中间2位月份、后2位日期，用"-"连接

### 4.2 数值类型转换

| 数据项 | 本公司API | 长软API | 转换说明 |
|--------|-----------|---------|----------|
| 金额 | `number` (18,2) | `VARCHAR2(32)` 字符串 | 长软→本公司: `parseFloat()` |
| 税额 | `number` (18,2) | `VARCHAR2(32)` 字符串 | 长软→本公司: `parseFloat()` |
| 价税合计 | `number` (18,2) | `VARCHAR2(32)` 字符串 | 长软→本公司: `parseFloat()` |
| 明细金额 | `number` (18,2) | `VARCHAR2(32)` 字符串 | 长软→本公司: `parseFloat()` |
| 明细税额 | `number` (18,2) | `VARCHAR2(32)` 字符串 | 长软→本公司: `parseFloat()` |

```
长软API返回：
```

## 六、增值税电子普通发票特殊性说明

### 6.1 该类型的独特特性

1. **电子发票介质**
   - 增值税电子普通发票是**纯电子介质**发票
   - 无需打印，通过电子邮件或网络下载获取
   - 长软API返回OFD和PDF文件下载链接（URL和PURL字段）
   - 电子文件与纸质发票具有同等法律效力

2. **有校验码**
   - 增值税电子普通发票**有校验码**字段（JYM）
   - 这是与专用发票的主要区别之一
   - 查验时需提供**校验码后6位**作为必填参数
   - 校验码用于税务系统真伪验证

3. **不可抵扣进项税**
   - 电子普通发票**不能抵扣进项税额**
   - 这是与电子专票的核心区别
   - 主要面向个人消费者或不需要抵扣的企业
   - 小规模纳税人可以开具电子普通发票

4. **购方信息可选**
   - 购方名称必填，但税号可以为空
   - 个人购买时，可以只有名称（如"个人"）
   - 地址电话、银行账号通常为空
   - 与专票要求完整购方信息不同

5. **销方可以是小规模纳税人**
   - 小规模纳税人可以开具电子普通发票
   - 自然人代开也可能是电子普通发票
   - 税率通常为3%或5%（小规模征收率），或13%、9%、6%（一般纳税人）
   - 一般纳税人也可开电子普票（如面向个人销售）

6. **必须有货物明细**
   - 电子普通发票必须包含货物或应税劳务、服务明细
   - CHILDLIST不能为空
   - 每个明细行需要包含完整的税率、税额信息

7. **文件下载链接**
   - 长软API返回电子发票文件下载链接
   - URL字段：OFD版式文件下载链接
   - PURL字段：PDF版式文件下载链接
   - 若为空则不能下载

8. **金额计算关系**
   ```
   价税合计 = 金额合计 + 税额合计
   税额合计 = Σ(明细金额 × 明细税率)
   金额合计 = Σ(明细金额)
   ```

### 6.3 特殊处理逻辑

#### 6.3.1 校验码提取与验证

**处理流程**：
1. **获取完整校验码**：从长软API的 `BODY/JYM` 字段获取
2. **提取后6位**：截取字符串最后6位字符
3. **查验时使用**：调用长软查验接口时，只需传入后6位
4. **返回完整校验码**：查验成功后，将完整校验码返回给本公司API

**映射逻辑**：
- 查验请求时：从用户输入获取校验码（可能是完整或后6位），如果超过6位，提取后6位
- 查验响应时：长软API返回完整校验码（`BODY/JYM`），直接映射到本公司API的 `verification_code` 字段

#### 6.3.2 购方信息可选处理

**挑战**：电子普票的购方信息可能不完整（如个人购买）。

**处理方案**：
1. **购方名称**：必填，如果是个人可以是"个人"、"*个人*"等
2. **购方税号**：可选，个人购买时为空
3. **购方地址电话**：可选，通常为空
4. **购方银行账号**：可选，通常为空

**映射逻辑**：
- 购方名称：直接映射，必填
- 购方税号：直接映射，可选（个人购买时为空或"*"）
- 购方地址电话：直接映射，可选（通常为空）
- 购方银行账号：直接映射，可选（通常为空）
- 判断是否为个人购买：如果购方税号为空或"*"，或购方名称包含"个人"，则is_personal_buyer = true

#### 6.3.3 电子文件下载链接处理

**挑战**：电子发票需要提供文件下载链接。

**处理方案**：
1. **URL字段**：OFD版式文件下载链接，若为空则不能下载
2. **PURL字段**：PDF版式文件下载链接，若为空则不能下载
3. 长软API返回这两个字段后，本公司API可以选择：
   - 直接透传给前端用户
   - 下载文件后存储到自己的服务器，返回自己的下载链接
   - 提供文件预览功能

**注意事项**：
- 下载链接可能有时效性，需要及时下载或转存
- OFD格式是国家标准的电子发票格式
- PDF格式是通用的文档格式，便于查看和打印

#### 6.3.4 小规模纳税人识别

**挑战**：需要识别销方是否为小规模纳税人。

**识别依据**：
1. **税率**：小规模征收率通常为3%或5%
2. **发票类型**：小规模可以开电子普票
3. **金额**：小规模纳税人季度销售额不超过30万免税

**映射逻辑**：
- 检查税率：如果税率为0.03或0.05，可能是小规模纳税人
- 检查所有明细税率：如果所有明细税率都是3%或5%或0，likely是小规模纳税人
- 结合发票类型：如果是电子普票且税率为3%或5%，seller_taxpayer_type_code = '2'（小规模纳税人）

#### 6.3.5 金额计算验证

**挑战**：确保明细数据与汇总数据一致。

**验证规则**：
1. **验证明细金额合计**：Σ(items.amount) ≈ amount (允许±0.02误差)
2. **验证明细税额合计**：Σ(items.tax_amount) ≈ tax_amount (允许±0.02误差)
3. **验证价税合计**：amount + tax_amount ≈ total_amount (允许±0.02误差)
4. **验证每个明细的计算逻辑**：item.tax_amount ≈ item.amount × item.tax_rate (允许±0.01误差)

**处理策略**：
- 计算明细金额合计
- 计算明细税额合计
- 对比汇总金额
- 如果误差超过阈值，记录警告日志，但不影响查验结果（以长软返回为准）

#### 6.3.6 税率识别与验证

**挑战**：电子普票的主税率需要从明细中提取。

**常见税率**：
- **13%**：一般货物（一般纳税人）
- **9%**：农产品、交通运输服务等（一般纳税人）
- **6%**：现代服务业、增值电信服务等（一般纳税人）
- **3%**：小规模纳税人征收率（最常见）
- **5%**：销售不动产、劳务派遣等小规模征收率
- **0%**：出口货物、免税商品等
- **免税**：特定商品

**提取逻辑**：
1. 遍历所有明细，提取非零税率
2. 如果所有明细税率一致，直接取该税率
3. 如果明细中有多种税率，优先选择最常见的税率，或取加权平均税率，或取第一个非零税率

### 7.4 增值税电子普通发票特殊集成要点

⚠️ **这是最重要的部分，必须根据业务特性详细说明！**

#### 7.4.1 校验码处理

**挑战**：电子普票查验必须提供校验码后6位，但长软返回完整校验码。

**解决方案**：
1. **查验请求时**：
   - 从用户输入获取校验码（可能是完整或后6位）
   - 如果超过6位，提取后6位
   - 调用长软API时传入后6位

2. **查验响应时**：
   - 长软API返回完整校验码（`BODY/JYM`）
   - 直接映射到本公司API的 `verification_code` 字段
   - 不需要截断或处理

**实现要点**：
- 用户输入校验码可能是完整32位或后6位，需要兼容处理
- 提取后6位：`last6 = jym.substr(jym.length - 6, 6)`
- 调用长软：`<JYM>后6位</JYM>`
- 响应映射：`verification_code = xmlBody.JYM`（完整校验码）

#### 7.4.2 电子文件下载链接处理

**挑战**：电子发票需要提供OFD和PDF文件下载链接。

**解决方案**：
1. **获取下载链接**：
   - 从长软API的 `BODY/URL` 字段获取OFD文件下载链接
   - 从长软API的 `BODY/PURL` 字段获取PDF文件下载链接
   - 若为空则不能下载

2. **处理策略**：
   - **策略1（透传）**：直接将长软的下载链接返回给前端用户
   - **策略2（转存）**：下载文件后存储到自己的服务器，返回自己的下载链接
   - **策略3（预览）**：提供文件预览功能，用户可在线查看

3. **扩展字段**：
   - 建议本公司API扩展 `ofd_file_url` 和 `pdf_file_url` 字段
   - 或者在响应中添加 `file_links` 对象，包含文件类型和下载链接

**注意事项**：
- 下载链接可能有时效性（如7天有效），需要及时下载或转存
- OFD格式是国家标准的电子发票格式，需要专用阅读器
- PDF格式是通用格式，便于查看和打印
- 如果采用转存策略，需要处理文件存储和管理

#### 7.4.3 购方信息可选处理

**挑战**：电子普票的购方信息可能不完整，尤其是个人购买。

**解决方案**：
1. **购方名称**：必填，但可以是"个人"、"*个人*"、"*"等
2. **购方税号**：可选，个人购买时为空或"*"
3. **购方地址电话**：可选，通常为空
4. **购方银行账号**：可选，通常为空

**业务处理**：
- 判断是否为个人购买：
  - 如果 `buyer_tax_no` 为空或"*"
  - 或 `buyer_name` 包含"个人"或等于"*"
  - 则 `is_personal = true`
- 个人购买时，`buyer_tax_no`、`buyer_address_phone`、`buyer_bank_account` 统一设为null
- 不要因为购方信息不完整而拒绝查验
- 只要长软查验成功，就应该返回成功

**注意事项**：
- 个人购买的发票完全合法有效
- 电子普票是面向个人消费者的主要发票类型
- 与纸质普票处理逻辑一致

#### 7.4.4 小规模纳税人识别

**挑战**：需要识别销方是否为小规模纳税人。

**识别依据**：
1. **税率为3%或5%**：小规模征收率
2. **只能开普票**：不能开专票
3. **发票类型为10**：电子普通发票

**业务处理**：
- 检查税率：`if (tax_rate === 0.03 || tax_rate === 0.05) { is_small_scale = true; }`
- 检查所有明细税率：所有明细税率都是3%或5%或0
- 如果满足条件：`seller_taxpayer_type_code = '2'`（小规模纳税人）

**注意事项**：
- 一般纳税人也可以开3%或5%税率的电子普票（简易计税）
- 最准确的识别需要从业务系统查询
- 仅供参考，不作为强制校验

#### 7.4.5 明细数据完整性验证

**挑战**：确保明细数据与汇总数据一致。

**验证规则**：
1. **验证明细金额合计**：`Σ(items.amount) ≈ amount` (允许±0.02误差)
2. **验证明细税额合计**：`Σ(items.tax_amount) ≈ tax_amount` (允许±0.02误差)
3. **验证价税合计**：`amount + tax_amount ≈ total_amount` (允许±0.02误差)
4. **验证每个明细的计算逻辑**：`item.tax_amount ≈ item.amount × item.tax_rate` (允许±0.01误差)

**处理策略**：
- 计算明细金额合计：`itemsAmount = Σ(item.JE)`，转换为number后求和
- 计算明细税额合计：`itemsTax = Σ(item.SE)`，转换为number后求和
- 对比汇总金额：`|itemsAmount - amount| <= 0.02`
- 对比汇总税额：`|itemsTax - tax_amount| <= 0.02`
- 验证价税合计：`|amount + tax_amount - total_amount| <= 0.02`
- 如果误差超过阈值：记录警告日志，但不影响查验结果（以长软返回为准）

**注意事项**：
- 由于浮点数计算精度问题，允许小额误差（通常2分钱以内）
- 如果误差超过阈值，应记录警告日志，但不影响查验结果
- 长软返回的明细字段金额、税额、税率都是字符串类型，需要转换为number后计算

#### 7.4.6 特殊票种处理

**成品油发票** (TSPZBZ=01):
- 明细中需要包含油品名称、单位、数量等信息
- 可能有特殊的格式要求
- 税率通常为13%

**农产品发票** (TSPZBZ=21):
- 税率通常为9%或10%
- 可能有免税或零税率

**稀土发票** (TSPZBZ=02):
- 稀土矿产品或稀土产成品
- 可能有特殊的税收政策
- 需要特殊标识和备案

**处理建议**：
- 根据 `TSPZBZ` 字段判断特殊票种
- 成品油发票：验证明细中是否有油品信息，检查税率是否符合成品油税率
- 农产品发票：检查税率是否为9%或10%
- 稀土发票：检查是否有稀土备案信息
